# 手势识别控制系统

## 项目简介

这是一个基于MediaPipe和OpenCV的手势识别控制系统，允许用户通过不同的手势来控制计算机执行特定操作。该项目使用计算机视觉和深度学习技术，通过摄像头实时捕获用户手势，并根据预设的手势模式执行相应的操作，如打开文件、执行Python脚本、访问网页等。

## 功能特点

- **实时手势识别**：通过摄像头实时捕获并识别用户手势
- **多种手势支持**：支持5种不同的手势（1-5根手指的不同组合）
- **可视化界面**：在屏幕上实时显示手部关键点、连接线和边界框
- **智能操作执行**：根据不同手势执行不同操作，如打开文件、执行脚本、访问URL等
- **错误处理机制**：包含完善的错误处理和依赖自动安装功能
- **手势冷却机制**：防止短时间内重复触发同一操作

## 系统要求

- Python 3.7+
- 摄像头设备
- Windows操作系统（推荐）

## 安装说明

1. 克隆或下载项目到本地
2. 安装必要的依赖库：

```bash
pip install opencv-python mediapipe
```

3. 准备配置文件：
   - 在项目目录下创建名为`address.DATA.txt`的文件
   - 在文件中添加5行内容，分别对应5种手势要执行的操作（文件路径、URL或Python脚本路径）

## 使用方法

1. 运行主程序：

```bash
python start_optimized.py
```

2. 程序启动后，将显示摄像头画面和实时手势识别结果
3. 做出不同的手势触发相应操作：
   - **1根手指**：执行配置文件中的第1行操作
   - **2根手指**：执行配置文件中的第2行操作
   - **3根手指**（特定组合）：执行配置文件中的第3行操作
   - **4根手指**：执行配置文件中的第4行操作
   - **5根手指**：执行配置文件中的第5行操作

4. 按空格键退出程序

## 配置说明

`address.DATA.txt`文件格式：
```
第1行：对应1根手指手势的操作路径
第2行：对应2根手指手势的操作路径
第3行：对应3根手指手势的操作路径
第4行：对应4根手指手势的操作路径
第5行：对应5根手指手势的操作路径
```

支持的操作类型：
- Python脚本文件（.py）：将自动执行
- 网址链接：将在浏览器中打开
- 其他文件：将使用系统默认程序打开

## 技术架构

- **MediaPipe**：用于手部和面部检测
- **OpenCV**：用于图像处理和可视化
- **Python**：主要编程语言
- **子进程管理**：用于执行外部程序和脚本

## 注意事项

1. 确保摄像头正常工作且光线充足
2. 手势应清晰可见，避免遮挡
3. 每次操作后有3秒的冷却时间，防止误触发
4. 执行Python脚本时，确保所需依赖已安装
5. 配置文件中的路径应正确且可访问

## 常见问题

**Q: 程序无法检测到手势？**
A: 确保光线充足，手部在摄像头视野内，且手势清晰可见。

**Q: 执行Python脚本失败？**
A: 检查脚本路径是否正确，所需依赖是否已安装。程序会提示自动安装常见依赖。

**Q: 手势识别不准确？**
A: 调整摄像头位置和角度，确保手部完整可见，避免快速移动。

**Q: 导入库失败，提示"找不到指定的程序"或"DLL load failed"？**
A: 这通常是因为您正在使用Python 3.14或更高版本，与某些库（如NumPy）不兼容。解决方案：
1. 推荐降级到Python 3.9或3.10版本
2. 如果必须使用Python 3.14，请按以下步骤操作：
   - 运行 `python install_dependencies.py` 安装兼容版本的依赖
   - 或手动执行：`pip install -r requirements.txt`
3. 如果仍然失败，可以尝试使用conda创建新环境：
   ```
   conda create -n gesture python=3.10
   conda activate gesture
   pip install -r requirements.txt
   ```
